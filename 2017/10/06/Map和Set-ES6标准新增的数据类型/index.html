Map和Set--ES6标准新增的数据类型

发表于 2017-10-06 | 分类于 技术

Map和Set是ES6标准新增的数据类型，请根据浏览器的支持情况决定是否要使用。

JavaScript的默认对象表示方式是对象 {}
对象有个小问题，就是键必须是字符串，但实际Number或者其他类型作为键也没毛病啊
为了解决这个问题，ES6引入了新的数据类型Map。
测试你的浏览器是否支持ES6规范，不支持报ReferenceError错误，不行就换

Map
键值对，查询速度快

一般方法先从names 找到对应位置，在从 scores取成绩，Array越长，耗时越长。
Map实现，只需 ‘名字-成绩’的对照表，表有多大都不会慢。

初始化Map需要 一个二维数据 || 初始化个空Map就ok

key只能对应一个value，故多次对一个key放入value，后把前覆盖掉。

Set
Set跟Map类似，也是一组key的集合，但不存储value. 由于key不能重复，故没重复的key。
